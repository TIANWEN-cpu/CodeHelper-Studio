# DevLearnerAI -- 下一轮冲刺计划

> 制定日期：2026-06-02
> 基于：HIGH_ROI_IMPROVEMENTS.md、SPRINT_REVIEW.md、当前任务清单
> 当前成熟度：L2 MVP | 目标：L2+ (用户体验和 AI 能力维度升级)
> 冲刺周期建议：2 周（10 个工作日）

---

## 一、冲刺背景

上一轮冲刺（Sprint 4-8 + Sprint 11+）完成了大量基础设施和性能优化工作，但受 API 429 限流影响，Sprint 11+ 任务完成率仅 40%。用户体验（L1）和 AI 能力（L1）两个最弱维度未得到实质改善，新用户首次体验仍然极差，统计数据仍存在丢失风险。

本轮冲刺聚焦于 **消灭 L1 维度** + **消除数据丢失风险**，不铺新功能，只把已搭好的框架做深做实。

---

## 二、冲刺目标（5 项，按优先级排序）

### 目标 1：统计数据迁移至 SQLite（P0 阻塞项）

| 维度         | 说明                                  |
| ------------ | ------------------------------------- |
| **优先级**   | P0 -- 数据安全                        |
| **预计工时** | 3 天                                  |
| **当前状态** | in_progress（方案设计完成，实现中断） |

**具体任务**

1. 审计 `StatsView.tsx` 中所有数据来源，列出仍在使用 localStorage 的读写点
2. 基于已有 `submissions`、`mistakes`、`chat_history` 表构建统计 SQL 查询：每日/每周做题量、通过率、按难度分布、AI 对话频次
3. 在 `electron/ipc/database.ts` 新增统计查询 IPC handler
4. 利用 `analytics_events` 表记录关键行为事件（题目开始、提交、AI 对话开始/结束）
5. 添加 JSON 格式数据导出功能，让用户可备份自己的学习数据
6. 清理所有 localStorage 业务数据读写，仅保留 UI 偏好（如主题、侧边栏宽度）

**成功标准**

- localStorage 中不再存储任何业务数据
- 清除浏览器缓存后统计数据完整保留
- 统计面板所有图表数据来自 SQLite 查询
- 支持 JSON 格式一键导出

**风险与缓解**

| 风险                       | 概率 | 缓解                                |
| -------------------------- | ---- | ----------------------------------- |
| 历史 localStorage 数据丢失 | 高   | 首次启动时检测并迁移旧数据到 SQLite |
| SQL 查询性能不足           | 中   | 对常用查询添加索引，结果缓存 5 分钟 |

### 目标 2：新用户引导流程上线（P1 最高 ROI）

| 维度         | 说明                                             |
| ------------ | ------------------------------------------------ |
| **优先级**   | P1 -- 用户体验 L1->L2 的关键                     |
| **预计工时** | 3 天                                             |
| **当前状态** | pending（onboarding store 已创建，未集成主流程） |

**具体任务**

1. 创建 `OnboardingWizard.tsx` 组件，实现 3 步引导：
   - **欢迎页**：产品定位一句话 + 核心功能预览（文字描述即可，无需截图）
   - **API Key 配置**：引导用户添加 AI 配置，提供"智能粘贴"解析 OpenAI 格式 URL
   - **第一道题**：自动选一道 easy 题目，引导完成"读题 -> 写代码 -> 运行"流程
2. 在 `App.tsx` 中检查 `settings` 表的 `onboarding_completed` 键，未完成时渲染向导
3. 完成后写入设置键，后续启动跳过
4. 在设置页面添加"重新运行入门引导"入口

**成功标准**

- 新用户首次打开可 2 分钟内完成引导
- 引导完成后自动进入第一道题目
- 已完成用户不再看到引导页
- 引导流程可在设置中重新触发

**风险与缓解**

| 风险                         | 概率 | 缓解                                        |
| ---------------------------- | ---- | ------------------------------------------- |
| 引导流程过长导致用户中途放弃 | 低   | 3 步精简设计 + "跳过" 按钮，总时长 < 2 分钟 |
| API Key 配置失败             | 中   | 提供"稍后配置"选项 + 清晰的错误提示         |

### 目标 3：AI 上下文感知注入（P1 产品核心差异化）

| 维度         | 说明                        |
| ------------ | --------------------------- |
| **优先级**   | P1 -- AI 能力 L1->L2 的关键 |
| **预计工时** | 3 天                        |
| **当前状态** | pending                     |

**具体任务**

1. 在 `src/stores/chatStore.ts` 的 `sendMessage` 方法中，读取 `problemStore` 和 `editorStore` 当前状态
2. 构建 context system message，自动注入：当前题目标题/描述/难度/标签、编辑器中的代码、最近提交结果
3. 在 `electron/ipc/ai.ts` 的 messages 数组头部插入 context message
4. 在设置中添加开关：用户可关闭自动上下文注入
5. 在 `ChatView.tsx` 中显示当前注入的上下文状态（如"正在讨论：两数之和"）

**成功标准**

- AI 回答中提及当前题目名称或代码变量名的比例 > 80%
- 用户手动描述题目的消息数量下降 > 50%
- 设置中可控制上下文注入开关

**风险与缓解**

| 风险                       | 概率 | 缓解                                                  |
| -------------------------- | ---- | ----------------------------------------------------- |
| 上下文注入增加 token 消耗  | 中   | 限制注入长度（题目描述截断 500 字，代码截断 2000 字） |
| 用户不想让 AI 看到当前代码 | 低   | 默认开启但提供关闭选项                                |

### 目标 4：C/C++/C# 评测诚实化（P0 发布阻塞项）

| 维度         | 说明                          |
| ------------ | ----------------------------- |
| **优先级**   | P0 -- 消除信任风险            |
| **预计工时** | 2 天                          |
| **当前状态** | in_progress（已开始，需完成） |

**具体任务**

1. 在 `EvaluationResult` 模型中新增 `evaluation_mode` 字段（`"runtime"` / `"structural"`）
2. 修改 `evaluate_keyword_code()` 返回的反馈文案为"结构检查通过/未通过（注意：当前语言仅检查代码结构，不验证编译和运行结果）"
3. 在 `practice.py` 练习详情面板中，当语言为 C/C++/C# 时显示黄色提示条
4. 添加 3-5 个单元测试验证新模式

**成功标准**

- C/C++/C# 练习评测结果明确标注"结构检查"而非"评测通过"
- 用户能清楚理解当前语言的评测局限性
- 新增测试全部通过

**风险与缓解**

| 风险                       | 概率 | 缓解                             |
| -------------------------- | ---- | -------------------------------- |
| 用户对"仅结构检查"失去兴趣 | 低   | 推荐 Python 练习获得完整评测体验 |
