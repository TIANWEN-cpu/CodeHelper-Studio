# 设置指南

CodeHelper 的设置面板集中管理所有可配置项，包括 AI 模型配置、主题切换、编辑器自定义等。

## 概览

设置模块（`SettingsView`）提供以下配置区域：

- AI 模型配置管理
- 编辑器设置
- 主题选择
- 通用设置

设置数据持久化存储在 SQLite 数据库的 `settings` 表中。

## AI 模型配置

### 添加新配置

1. 进入 **设置** 模块
2. 在 AI 配置区域点击 **添加配置**
3. 填写配置信息：

| 字段     | 必填 | 说明                         | 示例                        |
| -------- | ---- | ---------------------------- | --------------------------- |
| 名称     | 是   | 配置的显示名称               | `GPT-4o`                    |
| API Key  | 是   | API 密钥，加密存储           | `sk-xxx`                    |
| Base URL | 是   | API 服务基础地址（不含路径） | `https://api.openai.com/v1` |
| 模型     | 是   | 模型标识名称                 | `gpt-4o`                    |
| 设为默认 | 否   | 勾选后此配置为默认使用的模型 | -                           |
| 任务类型 | 否   | 用于区分不同场景的配置       | `chat`                      |

4. 点击 **保存**

### 编辑配置

- 在配置列表中点击编辑按钮
- 修改需要变更的字段
- 保存后立即生效

### 删除配置

- 在配置列表中点击删除按钮
- 确认后删除该配置
- 删除默认配置后，系统会自动选择第一个可用配置

### 设置默认配置

- 勾选某个配置的"设为默认"选项
- 同一时间只能有一个默认配置
- 未指定 `configId` 的 AI 调用会使用默认配置

### 获取可用模型

- 配置好 API Key 和 Base URL 后，系统调用 API 的 `/models` 接口
- 获取到的模型列表可用于快速选择模型名称
- 如获取失败，仍可手动输入模型名称

### API Key 安全

- API Key 使用 Electron 的 `safeStorage` API 加密
- 加密后的数据以 `enc:` 前缀标记，后接 Base64 编码的加密内容
- 如果系统不支持加密（极少数情况），以明文存储并输出警告日志
- 数据库中不会出现明文 API Key

## 主题设置

CodeHelper 内置三套主题配色方案：

### Catppuccin Mocha（默认）

| 用途     | 颜色值           |
| -------- | ---------------- |
| 主背景   | `#1e1e2e`        |
| 侧边栏   | `#181825`        |
| 强调色   | `#cba6f7` (紫色) |
| 正文     | `#cdd6f4`        |
| 次要文字 | `#6c7086`        |
| 成功     | `#a6e3a1`        |
| 警告     | `#f9e2af`        |
| 错误     | `#f38ba8`        |
| 信息     | `#89b4fa`        |

### Fjord

自然深沉的北欧峡湾色调，适合长时间编码。

### Ember

温暖的余烬色调，适合偏好暖色系的用户。

### 切换主题

在设置面板的主题区域选择即可切换，切换后立即全局生效。

## 编辑器设置

| 设置项   | 说明                    | 默认值 | 范围       |
| -------- | ----------------------- | ------ | ---------- |
| 字体大小 | Monaco 编辑器文字大小   | 14px   | 8px - 32px |
| Tab 宽度 | 每个 Tab 字符占用空格数 | 4      | 1 - 8      |

修改后对所有编辑器实例生效。

## 通用设置

### 打开外部链接

- 应用中的外部链接（如 GitHub 仓库链接）会在系统默认浏览器中打开
- 仅允许 `http:` 和 `https:` 协议的链接

### 数据管理

- 所有数据存储在本地 SQLite 数据库中
- 数据库位置因操作系统而异（参见 [快速开始 - 数据存储位置](getting-started.md#数据存储位置)）
- 手动备份：复制数据库文件即可

## 右键菜单

在应用中右键可弹出上下文菜单：

- 选中文本时：复制、全选
- 可编辑区域：粘贴、剪切、全选
- 其他区域：全选、复制

## 窗口行为

- 默认窗口大小：1200 x 800 像素
- 最小窗口大小：900 x 600 像素
- 背景色跟随当前主题
- macOS 上关闭所有窗口后应用不退出（可通过 Dock 重新打开）
- Windows/Linux 上关闭所有窗口后自动退出

## 性能监控

应用内置性能监控功能，每 5 分钟自动记录 IPC 调用统计信息，用于诊断性能问题：

- 记录各 IPC 通道的调用次数和平均耗时
- 识别慢操作（超过阈值的调用）
- 日志输出到 Main 进程的控制台
