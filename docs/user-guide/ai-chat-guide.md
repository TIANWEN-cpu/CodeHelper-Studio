# AI 对话指南

CodeHelper 内置 AI 编程助手，支持 OpenAI 兼容 API，提供流式对话、预设提示词和长期记忆系统。

## 概览

AI 对话模块由以下部分组成：

- **ChatView** - 对话主界面
- **SessionList** - 会话列表侧栏
- **MessageBubble** - 消息气泡组件（支持 Markdown 渲染与代码高亮）

## 开始对话

### 前置条件

在使用 AI 对话之前，需要先在 **设置** 中配置至少一个 AI 模型（详见 [快速开始 - API 配置](getting-started.md#api-配置)）。

### 创建新会话

1. 点击左侧栏的 **AI助手** 图标进入对话界面
2. 点击会话列表上方的 **新建会话** 按钮
3. 在输入框中输入你的问题或指令
4. 按回车键或点击发送按钮

### 发送消息

- 在底部输入框输入内容，支持多行文本
- 按 `Enter` 发送消息
- 按 `Shift+Enter` 换行
- 消息发送后，AI 将以流式输出方式逐字符渲染回复

## 会话管理

### 会话列表

- 所有会话按最后更新时间倒序排列
- 点击会话标题切换到对应会话
- 当前会话会有高亮标识

### 会话操作

| 操作           | 方式                                     |
| -------------- | ---------------------------------------- |
| 新建会话       | 点击会话列表上方的新建按钮               |
| 切换会话       | 点击会话列表中的会话标题                 |
| 重命名         | 双击或右键会话标题进行编辑               |
| 删除会话       | 右键菜单选择删除（同时删除所有聊天记录） |
| 设置系统提示词 | 在会话设置中配置该会话的 system prompt   |

### 系统提示词 (System Prompt)

每个会话可以配置独立的系统提示词，用于指导 AI 的行为模式：

- 点击会话设置或编辑图标
- 输入系统提示词内容
- AI 在该会话的所有对话中都会参考此提示词

## 预设提示词系统

CodeHelper 内置多套预设提示词模板，同时支持自定义。

### 使用预设提示词

1. 在对话界面中找到预设提示词入口
2. 从列表中选择一个预设模板
3. 模板内容将填充到输入框，可根据需要修改后发送

### 管理预设提示词

- **查看内置预设**：预设列表中带有"内置"标记的不可编辑或删除
- **创建自定义预设**：输入名称和提示词内容后保存
- **编辑预设**：仅自定义预设可编辑，内置预设为只读
- **删除预设**：仅自定义预设可删除

### 预设提示词示例

- 代码解释 - 要求 AI 详细解释代码逻辑
- 代码审查 - 请求代码审查与改进建议
- Bug 修复 - 描述问题并请求修复方案
- 算法分析 - 分析算法的时间复杂度和空间复杂度

## 长期记忆系统

长期记忆系统允许 AI 跨会话记住你的偏好和重要信息。

### 自动记忆

AI 会自动从对话中提取关键信息并保存为记忆：

- 用户的编程偏好和习惯
- 重要的上下文信息
- 明确表达的偏好或要求

### 手动管理记忆

| 操作     | 说明                         |
| -------- | ---------------------------- |
| 查看记忆 | 在设置中浏览所有已保存的记忆 |
| 新建记忆 | 手动添加一条新的记忆         |
| 编辑记忆 | 修改记忆内容或分类           |
| 删除记忆 | 移除不再需要的记忆           |
| 置顶记忆 | 标记为始终注入到对话中       |
| 禁用记忆 | 暂停某条记忆的使用而不删除   |

### 记忆注入机制

- 对话时，系统会根据当前消息内容匹配相关记忆
- 置顶记忆始终会被注入
- 相关记忆以 system 消息的形式插入到对话上下文中
- 记忆匹配基于关键词相似度，最近使用的记忆优先

## Markdown 渲染

AI 回复中的 Markdown 内容会被自动渲染，包括：

- **标题** - 多级标题展示
- **代码块** - 语法高亮显示，支持复制
- **行内代码** - 灰底高亮展示
- **列表** - 有序和无序列表
- **表格** - 结构化数据展示
- **粗体/斜体** - 文本格式化
- **链接** - 可点击的超链接
- **删除线** - 标记删除的文本
- **引用** - 块引用样式
- **任务列表** - 带勾选框的列表

## 对话技巧

### 编程问题

- 明确描述你遇到的问题
- 提供相关的代码片段
- 说明期望的行为和实际行为
- 指定编程语言和运行环境

### 代码生成

- 详细描述功能需求
- 指定编程语言和框架
- 如有特定的代码风格要求请说明
- 可以要求 AI 分步实现

### 代码审查

- 将代码粘贴到对话中
- 要求 AI 从特定角度审查（性能、安全、可读性等）
- 请求改进建议和替代方案

### 调试帮助

- 提供完整的错误信息
- 说明复现步骤
- 列出已经尝试的解决方法

## 配置选项

### 选择模型

- 对话时可选择使用不同的 AI 模型
- 通过设置中的 AI 配置管理多个模型
- 不同模型适合不同任务（代码生成、文本分析等）

### 流式输出

- AI 回复默认以流式方式输出
- 逐字符渲染，实时看到回复内容
- 长回复不会阻塞界面

## 注意事项

- 每次对话的消息数量上限为 200 条
- 单条消息内容上限为 100,000 字符
- 系统提示词上限为 10,000 字符
- AI 调用会消耗 API 服务的额度，请合理使用
- 部分 API 服务可能需要网络代理才能访问
