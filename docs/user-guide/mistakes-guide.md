# 错题本指南

错题本自动追踪你在题库系统中的失败提交，帮助你分析薄弱环节并有针对性地强化练习。

## 概览

错题本模块提供：

- 从失败的代码提交中自动收集错题
- 追踪错误次数与错误类型
- 记录错误代码和正确代码
- 支持 AI 分析薄弱知识点
- 一键重做错题

## 错题收集

### 自动收集机制

当你在题库系统中提交代码并被判为失败时，系统会自动执行以下操作：

1. **新建错题**：如果该题目首次失败，在错题本中创建一条新记录
2. **更新错题**：如果该题目已有记录，更新错误次数和最新错误代码
3. **记录正确代码**：当你通过该题目时，自动保存你的正确代码

### 触发条件

以下判题结果会触发错题记录：

| 判题结果   | 说明                   |
| ---------- | ---------------------- |
| 答案错误   | 输出结果与预期不匹配   |
| 编译错误   | 代码无法通过编译       |
| 运行时错误 | 程序运行过程中发生异常 |
| 超时       | 程序运行超过时间限制   |

> 通过 (accepted) 的提交不会记录为错题。

## 浏览错题

### 错题列表

点击左侧栏的 **错题本** 图标查看所有错题。列表展示：

- **题目标题** - 关联的题目名称
- **难度** - 简单 / 中等 / 困难
- **标签** - 题目所属的知识点标签
- **错误次数** - 该题目的累计失败次数
- **错误类型** - 历史出现过的错误类型汇总
- **最后更新时间** - 最近一次出错的时间

### 错题详情

点击错题条目可查看详细信息：

- **题目描述** - 完整的问题描述
- **最后错误代码** - 最近一次失败提交的代码
- **正确代码** - 你曾经通过该题目的代码（如有）
- **AI 分析** - AI 对错误的分析（如有）

## 错误类型

错题本追踪以下四种错误类型：

| 错误类型   | 英文标识        | 含义                   |
| ---------- | --------------- | ---------------------- |
| 编译错误   | `compile_error` | 代码语法或编译问题     |
| 运行时错误 | `runtime_error` | 空指针、越界等运行异常 |
| 答案错误   | `wrong_answer`  | 逻辑错误导致输出不正确 |
| 超时       | `timeout`       | 算法效率不足           |

- 同一题目可能多次出现不同类型的错误
- 错误类型以 JSON 数组的形式存储在 `error_types` 字段中
- 重复的错误类型不会被重复记录

## AI 分析功能

### 获取 AI 分析

1. 打开错题详情
2. 点击 **AI 分析** 按钮
3. 系统将题目描述和你的错误代码发送给 AI
4. AI 返回分析结果，包括：
   - 错误原因诊断
   - 代码中的问题点
   - 改进建议
   - 相关知识点说明

### 分析结果存储

- AI 分析结果保存在数据库中（`ai_analysis` 字段）
- 分析内容上限为 50,000 字符
- 再次请求分析会覆盖之前的分析结果
- 即使关闭应用，分析结果也不会丢失

## 重做错题

### 操作步骤

1. 在错题本中找到要重做的题目
2. 点击 **重做** 按钮
3. 系统跳转到题库的题目详情页面
4. 编写新的解答代码并提交
5. 如果通过，正确代码自动保存到错题本

### 利用正确代码

- 如果之前已经通过该题目，错题本中会保存正确代码
- 可以查看正确代码作为参考
- 对比错误代码和正确代码，理解问题所在

## 删除错题

- 在错题列表中选择要删除的错题
- 点击删除按钮
- 错题记录将被永久删除

## 使用建议

### 定期复习

- 定期查看错题本，回顾曾经出错的题目
- 优先关注错误次数多的题目
- 注意不同类型错误的分布，了解自己的薄弱环节

### 分析错误模式

- **频繁的编译错误**：可能是语言基础不扎实，建议复习语法
- **频繁的运行时错误**：注意边界条件和空值处理
- **频繁的答案错误**：可能是算法理解不够深入
- **频繁的超时**：需要学习更高效的算法

### 配合 AI 使用

- 让 AI 分析你的错误代码，找出问题根源
- 请求 AI 推荐相关的学习资料
- 让 AI 给出优化建议和替代方案
