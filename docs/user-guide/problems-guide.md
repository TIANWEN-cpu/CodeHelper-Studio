# 题库指南

CodeHelper 内置 158+ 道编程题目，覆盖力扣、牛客、PAT、CSP、数学建模等多个来源，支持六种编程语言的自动判题。

## 概览

题库模块由以下组件构成：

- **ProblemsView** - 题库主界面
- **ProblemList** - 题目列表（支持筛选）
- **ProblemDetail** - 题目详情与作答
- **AISidebar** - AI 辅助解题侧栏

## 浏览题目

### 题目列表

点击左侧栏的 **刷题** 图标进入题库界面。题目以列表形式展示，每条显示：

- 题目标题
- 难度标签（简单 / 中等 / 困难）
- 来源标记
- 解题状态（已通过 / 未做）

### 筛选与排序

支持多维度筛选，帮助你快速找到目标题目：

| 筛选维度 | 可选值                                           |
| -------- | ------------------------------------------------ |
| 难度     | `easy`（简单）、`medium`（中等）、`hard`（困难） |
| 标签     | 数组、字符串、动态规划、树、图等                 |
| 来源     | `basic`、`leetcode`、`math-modeling` 等          |
| 赛道     | 按学习路径分类                                   |
| 平台     | `internal`、`leetcode`、`nowcoder` 等            |
| 模式     | `oj`（在线判题）等                               |
| 状态     | 已解决 / 未解决                                  |

### 题目详情

点击题目标题进入详情页面，包含：

- **题目描述** - 完整的问题描述，支持 Markdown 渲染
- **示例** - 输入输出示例
- **初始代码** - 各语言的代码模板
- **测试用例** - 题目附带的测试用例
- **提交历史** - 该题目的所有提交记录

## 解题与提交

### 选择编程语言

CodeHelper 支持以下六种编程语言：

| 语言       | 标识     | 编译器/运行时  |
| ---------- | -------- | -------------- |
| Python     | `python` | python         |
| C          | `c`      | gcc            |
| C++        | `cpp`    | g++            |
| C#         | `csharp` | csc/dotnet     |
| JavaScript | `js`     | node           |
| SQL        | `sql`    | 内置内存数据库 |

### 编写代码

1. 在题目详情页面，选择编程语言
2. 初始代码模板自动填充到编辑器
3. 在编辑器中编写你的解答代码

### 判题流程

1. 点击 **提交** 按钮
2. 系统逐用例运行你的代码
3. 实时显示每条用例的运行结果
4. 最终返回判定结果

### 判题结果

| 状态       | 英文标识        | 说明                         |
| ---------- | --------------- | ---------------------------- |
| 通过       | `accepted`      | 所有测试用例均通过           |
| 答案错误   | `wrong_answer`  | 输出结果与预期不匹配         |
| 编译错误   | `compile_error` | 代码无法通过编译（C/C++/C#） |
| 运行时错误 | `runtime_error` | 程序在运行过程中发生异常     |
| 超时       | `timeout`       | 程序运行超过 10 秒限制       |

### 提交记录

- 每次提交自动保存到数据库
- 记录信息包括：提交代码、语言、状态、通过用例数、总用例数、运行时间
- 在题目详情页面可查看该题目的所有提交记录（最近 20 条）

## SQL 题目特殊说明

SQL 题目使用内置的内存 SQLite 数据库执行：

- 代码中可以包含多条 SQL 语句
- 最后一条语句如果是查询语句（SELECT），其结果将作为输出
- 判题时对输出进行规范化比较
- 不需要外接数据库即可运行

## AI 辅助解题

题目详情页面内置 AI 侧栏，可随时请求 AI 帮助：

- **理解题意** - 让 AI 解释题目的要求
- **提示思路** - 获取解题方向的提示
- **代码审查** - 让 AI 审查你的代码
- **复杂度分析** - 分析算法的时间和空间复杂度
- **优化建议** - 获取代码优化方向

使用方法：

1. 在题目详情中点击 AI 侧栏图标
2. 输入你的问题（或直接引用题目上下文）
3. AI 会结合题目信息给出针对性的解答

## 题库数据来源

题目数据存储在 `resources/problems/` 目录下：

| 文件                 | 来源     | 题目数 |
| -------------------- | -------- | ------ |
| `basic.json`         | 基础练习 | 48 道  |
| `leetcode.json`      | 力扣经典 | 80 道  |
| `math-modeling.json` | 数学建模 | 30 道  |

### 题目数据同步

- 应用启动时自动从 JSON 文件同步题目到数据库
- 新题目自动插入，已有题目更新内容
- 同步依据为 `title + source` 的唯一组合

## 题目元数据

每道题目包含以下元数据：

| 字段             | 说明                                   |
| ---------------- | -------------------------------------- |
| `title`          | 题目标题                               |
| `description`    | 题目描述（Markdown 格式）              |
| `difficulty`     | 难度：easy / medium / hard             |
| `tags`           | 标签数组（如数组、字符串、动态规划等） |
| `languages`      | 支持的编程语言列表                     |
| `examples`       | 示例输入输出                           |
| `test_cases`     | 自动判题用的测试用例                   |
| `starter_code`   | 各语言的初始代码模板                   |
| `source`         | 来源标识                               |
| `tracks`         | 所属赛道                               |
| `platform`       | 平台标识                               |
| `mode`           | 题目模式（如 oj）                      |
| `exam_style`     | 考试风格（如 acm）                     |
| `estimated_time` | 预估完成时间                           |

## 最佳实践

1. **先理解再编码** - 仔细阅读题目描述和示例
2. **本地测试** - 先在编辑器中运行代码确认输出
3. **查看用例** - 留意题目中的边界用例
4. **利用 AI** - 遇到困难时不要犹豫使用 AI 侧栏
5. **回顾错题** - 错误的题目会自动进入错题本，定期复习
