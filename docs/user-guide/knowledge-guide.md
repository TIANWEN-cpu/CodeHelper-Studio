# 知识库指南

CodeHelper 内置知识库 RAG（检索增强生成）系统，支持导入本地文档并进行关键词检索，为 AI 对话提供参考上下文。

## 概览

知识库模块提供：

- 文档导入（支持 PDF、Markdown、TXT）
- 自动文本分块（约 500 字符/块）
- 关键词匹配检索，按相关度排序
- 向量嵌入字段已预留，支持未来升级为语义检索

## 导入文档

### 支持的文件格式

| 格式     | 扩展名 | 最大大小 | 说明             |
| -------- | ------ | -------- | ---------------- |
| 文本文件 | `.txt` | 10 MB    | 纯文本格式       |
| Markdown | `.md`  | 10 MB    | Markdown 格式    |
| PDF      | `.pdf` | 10 MB    | 自动提取文本内容 |

### 导入步骤

1. 点击左侧栏的 **知识库** 图标
2. 点击 **导入文档** 按钮
3. 在弹出的文件选择对话框中选择文件（支持多选）
4. 系统自动处理文件：
   - 读取文件内容
   - 对于 PDF 文件，使用 `pdf-parse` 库提取文本
   - 将文本按约 500 字符自动分块
   - 将文档元数据和分块数据存入数据库
5. 导入完成后，文档出现在知识库列表中

### 文档列表

导入后的文档在知识库列表中展示：

- 文件名
- 文件类型（txt / md / pdf）
- 分块数量
- 导入时间

## 检索文档

### 关键词检索

1. 在知识库界面的搜索框中输入关键词
2. 支持多关键词搜索（空格分隔）
3. 系统在所有文档的分块中匹配关键词
4. 结果按关键词匹配频率排序
5. 返回最相关的前 5 条结果

### 检索机制

- 检索范围：所有已导入文档的分块内容
- 匹配方式：大小写不敏感的关键词包含匹配
- 排序依据：关键词在文本中的出现次数总和
- 结果限制：单次最多返回 5 条匹配结果
- 关键词过滤：长度 <= 1 的关键词会被忽略

### 检索结果

每条检索结果包含：

- 匹配的文本片段（分块内容）
- 来源文件名
- 相关度评分

## 删除文档

- 在知识库列表中选择要删除的文档
- 点击删除按钮
- 文档及其所有分块数据将被级联删除

## 使用场景

### 学习笔记检索

将你的学习笔记、课程资料导入知识库，需要时快速检索相关内容：

- 编程语言教程
- 算法笔记
- 面试准备资料

### 代码文档检索

导入常用的技术文档，在编码时快速查阅：

- API 文档
- 框架使用指南
- 最佳实践文档

### 搭配 AI 使用

知识库检索结果可以作为 AI 对话的上下文参考：

1. 先在知识库中检索相关内容
2. 将检索结果粘贴到 AI 对话中
3. 基于文档内容向 AI 提问

## 技术细节

### 文本分块算法

文本分块使用以下策略：

- 默认块大小：约 500 字符
- 在自然断点处分割（段落、句子边界）
- 保留分块的顺序索引信息
- 每个分块独立存储，支持精确检索

### 存储结构

知识库使用两张数据库表：

- `knowledge_docs` - 存储文档元数据和原始内容
- `knowledge_chunks` - 存储文档分块，通过 `doc_id` 关联到文档

### 未来扩展

知识库系统已预留向量嵌入字段（`embedding`），支持未来升级为语义检索模式，届时将实现：

- 基于语义相似度的检索
- 更精确的上下文匹配
- AI 驱动的智能摘要
