## 五、用户影响评估

### 正面影响

- **首次体验改善:** 欢迎向导 + 功能导览 + 设置检查清单，新用户从"不知道做什么"到"有引导完成首次配置" -- 预计首次留存率提升 30-50%
- **AI 能力增强:** 代码分析器 + 代码解释按钮 + AI 上下文帮助，AI 从"纯聊天"到"有编程感知" -- 预计 AI 使用频率提升
- **性能提升:** 启动速度、内容加载、仪表板查询均有改善
- **稳定性:** 1000+ 测试用例 + 集成测试建立了质量安全网
- **安全性:** 沙箱加固、HTTPS 强制、密钥管理 -- 用户代码执行更安全

### 负面影响 / 用户感知不足

- **C/C++/C# 评测仍为关键字检查:** 诚实化 UI 未接入，用户可能误以为评测结果可靠
- **AI 仍需用户自带 API Key:** 核心门槛未消除，90% 目标用户无法使用 AI 功能
- **无安装包:** 需 Python 3.9+ + pip install + 命令行启动，非开发者无法完成
- **仪表盘仍为空白:** demo 数据和 analytics 集成未完成，新用户仍看到空仪表盘
- **内容无交互:** 课程仍为纯 Markdown 阅读，无代码运行、无测验、无检查点

### 用户需感知的 Breaking Changes

- **模块拆分:** `ai_mentor.py` 和 `practice_service.py` 已拆分为子包，直接导入旧路径的脚本需要更新 import 路径 (内部调用保持向后兼容)
- **构建脚本变更:** 旧的 `build_exe.py` 等 3 个脚本已删除，需使用 `scripts/build/build.py`

## 六、经验教训

### 做得好的

1. **架构重构彻底:** 1300 行巨型模块拆分为清晰子包，每个文件 < 300 行，可维护性大幅提升
2. **测试质量高:** 1000+ 测试覆盖安全、数据库、评测、AI、集成等多个层面，安全测试尤为全面
3. **安全意识强:** 沙箱 AST 预检、HTTPS 强制、keyring 密钥管理，安全水平超出同类项目
4. **Sprint 11 UX 改善到位:** 欢迎向导 + 功能导览 + 设置检查清单，直击"首次体验 L1"的核心痛点
5. **AI 功能初步赋能:** 代码分析器和代码解释按钮，让 AI 从"纯聊天"向"编程感知"迈出第一步

### 需要改进的

1. **Sprint 14 发布准备执行力不足:** 受 429 限流影响，发布准备几乎未完成，暴露了自动化流程的脆弱性
2. **同时并行 4 个 Sprint (11-14):** 多 Sprint 并行导致每个 Sprint 都无法完整收尾，建议未来按序执行
3. **C/C++/C# 评测诚实化被搁置:** ROI 最高的改进项 (8.75) 仍未完成，用户体验风险持续存在
4. **国际化"做了但没做完":** 有框架无内容，不如不做 -- 给用户一个"看起来支持但实际不工作"的选项比没有更糟
5. **测试覆盖高但功能深度低:** 1000 测试 vs 45% 产品完成度，说明测试投入与功能投入的比例失衡

### 关键决策

1. **Sprint 并行策略需要调整:** 4 个 Sprint 同时进行导致 4 个都未完成，未来应串行执行，每 Sprint 2-3 周
2. **API 限流保护机制必须建立:** 自动化流程的韧性设计应作为基础设施优先建设
3. **下一冲刺应聚焦发布阻塞项:** C/C++/C# 诚实化、安装包、空状态改造、GitHub 社区基础设施
