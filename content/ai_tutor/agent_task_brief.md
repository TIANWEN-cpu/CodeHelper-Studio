# Agent 任务说明书：让目标可执行

当你把一句“帮我优化页面”交给 Agent，它很容易误解范围。更稳的做法是先写一份任务说明书，把目标、上下文、约束、验收标准和风险点放在一起。

## 四段结构

1. **目标**：要完成什么，最好能用一句话判断成功。
2. **上下文**：当前页面、文件、数据或错误信息。
3. **约束**：不能改什么、要沿用什么风格、是否需要测试。
4. **验收**：完成后如何证明它真的可用。

> 好的 Agent 任务不是更长，而是更容易验证。

## 示例模板

| 字段   | 示例                                       |
| ------ | ------------------------------------------ |
| 目标   | 修复 AI 侧边栏遮挡主页面的问题             |
| 上下文 | 当前布局应像 VS2022 Copilot 一样并列排版   |
| 约束   | 不重置用户已有改动，保留旧 AI 侧边栏功能   |
| 验收   | 打开侧边栏后主页面宽度缩小，拖动宽度可保存 |

```md
目标：让 AI 助手侧边栏与主页面并列显示
上下文：用户希望侧边栏从右侧弹出，但不能覆盖主页面
约束：保留现有 AI Tutor 对话能力和路由入口
验收：浏览器中拖动侧边栏后 localStorage 保存宽度
```

## 使用检查

- 任务是否能拆成 3 到 6 步？
- 每一步是否有明确输入和输出？
- 是否需要人工确认高风险动作？
- 是否写清楚失败时要回滚还是暂停？

## 小练习

为“扩展 AI Tutor 课程内容”写一份 Agent 任务说明书，要求包含课程主题、目标读者、Markdown 格式和验收方式。
