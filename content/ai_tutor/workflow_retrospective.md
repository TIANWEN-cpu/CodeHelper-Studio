# Workflow 复盘：把一次成功变成流程

一次成功的 AI 协作如果没有复盘，下次还会重新摸索。Workflow 的价值，是把“这次怎么做成的”沉淀成可重复的步骤。

## 复盘五问

1. **目标**：这次解决的真实问题是什么？
2. **输入**：用到了哪些上下文、文件、日志或截图？
3. **步骤**：哪些动作是必要的，哪些只是试探？
4. **确认点**：哪些地方需要人类判断或授权？
5. **验收**：最终用什么测试、构建或浏览器操作证明完成？

## Workflow 模板

```md
# <Workflow 名称>

## 目标

## 输入

## 步骤

## 人类确认点

## 验收

## 下次优化
```

| 阶段 | 产物                 |
| ---- | -------------------- |
| 分析 | 问题边界和风险       |
| 执行 | 小步代码改动         |
| 验证 | 命令输出和浏览器证据 |
| 复盘 | 可复用模板           |

> Workflow 不追求一次写完，追求每轮都比上一轮更清晰。

## 小练习

把“AI 侧边栏并列排版 + 可拖拽宽度”整理成一个 Workflow，标出至少一个人类确认点和两个验收动作。
