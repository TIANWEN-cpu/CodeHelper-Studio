# 工具安全清单：让 AI 操作可控

AI 助手一旦能读写文件、运行命令、调用接口，就不再只是聊天窗口。它需要明确的风险分级、确认点和回滚策略。

## 风险分级

| 风险 | 操作                                       | 处理方式                       |
| ---- | ------------------------------------------ | ------------------------------ |
| 低   | 读取文件、查看日志、运行只读搜索           | 可以直接执行并记录证据         |
| 中   | 修改源码、更新配置、运行格式化             | 执行前说明范围，执行后验证     |
| 高   | 删除文件、上传数据、发布版本、调用外部服务 | 必须让用户确认或拆成更安全步骤 |

> 不确定风险时，把它当成更高一级处理。

## 确认文案

确认文案要说清三件事：

1. **动作**：将要做什么。
2. **目标**：作用在哪个文件、目录或服务。
3. **影响**：可能改变什么状态。

```txt
确认: 写入文件 -> src/App.tsx
影响: 会改变主界面布局，需要随后运行 typecheck 和浏览器验证。
```

## 回滚预案

- 修改前确认 git 状态，区分用户已有改动。
- 优先小步修改，避免一次性重写大文件。
- 失败后记录错误输出和已改文件。
- 对高风险操作保留备份或让用户确认。

## 小练习

实现 `tool_risk(action)`，把 `read` 判为 `low`，`write` 判为 `medium`，`delete` 和 `upload` 判为 `high`。
