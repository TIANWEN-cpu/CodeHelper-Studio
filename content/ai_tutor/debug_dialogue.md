# 和 AI 一起调试：从报错到假设

调试时不要只把一句“代码报错了”丢给 AI。更好的方式是让 AI 帮你建立假设，再逐步验证。

## 最小调试包

| 内容     | 说明                         |
| -------- | ---------------------------- |
| 报错原文 | 复制关键堆栈，不要只描述感觉 |
| 复现步骤 | 从输入到触发错误的最短路径   |
| 当前假设 | 你认为可能错在哪里           |
| 期望行为 | 正确结果应该是什么           |

## 对话顺序

1. 让 AI 先复述问题，确认它理解了现象。
2. 让 AI 列出 3 个可能原因，并按概率排序。
3. 每次只验证一个假设。
4. 修复后补测试，防止回归。

```python
def divide(a, b):
    return a / b

# 如果 b 可能是 0，应该先设计输入校验或异常处理。
```

> AI 给出的修复建议必须经过运行、测试或代码审查验证。
